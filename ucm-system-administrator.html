02 — Challenges & Response
Problems Encountered. Fixes Deployed.
Each challenge maps directly to a JD requirement for this role, drawn from production experience at Infosys managing enterprise Linux and AWS environments.
Challenge
1
Manual server monitoring with no automated alerting

Production servers had no proactive incident detection. Issues surfaced only after user impact.

2
Access control gaps and inconsistent RBAC across systems

Users held excess permissions across multiple systems; no unified PII audit trail existed.

3
Slow backup validation and no DR testing cadence

Backups were created but restore procedures were untested, creating unquantified recovery risk.

4
Poor technical documentation, long onboarding for new engineers

No standardized runbooks. Every new team member required weeks of tribal knowledge transfer.

Response
1
Python + CloudWatch automated alerting pipeline

Built threshold-based alerting on server metrics. Result: 85% reduction in production incidents reaching users.

2
RBAC framework with PII controls and audit logging

Implemented role-based access tiers and automated audit logging across all layers. Result: 100% compliance on audit review.

3
Airflow-scheduled backup validation and restore drills

Automated restore testing cadence with S3 versioned snapshots. Result: 99% uptime SLA maintained across all critical systems.

4
Confluence runbooks and technical SOPs

Authored 15+ standardized runbooks covering top server types and incident procedures. Result: 40% reduction in onboarding time.
